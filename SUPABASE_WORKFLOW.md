# Supabase Database Workflow

Этот документ описывает, как управлять базой данных в проекте Grocery App, сохраняя чистоту и порядок.

## 1. Структура папки `supabase/`

- `schema.sql` — **Главный источник правды**. Содержит полную структуру всей базы (таблицы, RLS, функции). Используется для развертывания проекта с нуля.
- `seed.sql` — Данные для первоначального наполнения.
- `admin_setup.sql` — Скрипт быстрой настройки прав администратора.
- `migrations/` — (Создается CLI) Папка для временных файлов изменений.

## 2. Как вносить изменения (Миграции)

Если нужно добавить новое поле или таблицу:

1. **Создайте файл миграции**:
   ```bash
   npx supabase migration new add_my_new_feature
   ```
2. **Напишите SQL** в созданном файле в папке `supabase/migrations/`.
3. **Примените к живой базе**:
   ```bash
   npx supabase db push
   ```
4. **Обновите `schema.sql`**:
   Скопируйте изменения из миграции в основной файл `schema.sql`, чтобы он оставался актуальным.

## 3. Очистка и Синхронизация

Раз в несколько недель или после завершения крупной задачи:
1. Убедитесь, что все изменения отражены в `schema.sql`.
2. Перенесите старые файлы миграций в `archive/` или удалите их, если они уже зафиксированы в `schema.sql`.
3. Сверяйте локальный `schema.sql` с живой базой через `db dump` (если установлен Docker).

---
*Следуйте этому процессу, чтобы избежать хаоса в структуре данных.*

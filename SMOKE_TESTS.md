# Smoke Tests / Чеклист приёмочного тестирования

> Использовать после серьёзных изменений в коде (рефакторинг, миграции, обновление зависимостей).
> Помечать `[PASS]` / `[FAIL]` / `[SKIP]`.

---

## 1. Авторизация и сессия

### 1.1 Вход по номеру телефона
- [ ] Экран логина открывается без краша
- [ ] Ввод номера телефона → приходит SMS с OTP
- [ ] Ввод корректного OTP → редирект на главный экран (Home)
- [ ] Ввод неверного OTP → отображается ошибка, повторная попытка доступна
- [ ] Повторный запрос OTP (кнопка «Отправить повторно») → новый код приходит

### 1.2 Сессия и её восстановление
- [ ] После перезапуска приложения пользователь остаётся залогиненным
- [ ] При истёкшем/невалидном токене — автоматический разлогин + редирект на логин
- [ ] `AuthProvider` корректно переключает `loading` → `false` (нет бесконечного спиннера)
- [ ] Телефон пользователя автоматически синхронизируется с `profiles` после логина

### 1.3 Выход из аккаунта
- [ ] Кнопка «Выйти» в профиле → сессия очищается, редирект на логин
- [ ] После выхода корзина и избранное сбрасываются (или корректно обнуляются)

## 2. Каталог и навигация

## 3. Корзина и ценообразование

### 3.1 Управление корзиной
- [ ] Добавление товара → позиция появляется в корзине, бейдж обновляется
- [ ] Увеличение количества → `totalItems` и `subtotal` пересчитываются
- [ ] Уменьшение количества до 0 → позиция удаляется из корзины
- [ ] Удаление позиции свайпом/кнопкой → корзина обновляется
- [ ] Batch-добавление (повтор заказа) → все позиции корректно добавляются

### 3.2 Логика цен и доставки
- [ ] `subtotal` = Σ(цена × количество) по всем позициям
- [ ] Корзина < 700₽ → `deliveryFee` = 90₽
- [ ] Корзина ≥ 700₽ → `deliveryFee` = 0₽
- [ ] Пустая корзина → `deliveryFee` = 0₽, `totalPrice` = 0₽
- [ ] `totalPrice` = `subtotal` + `deliveryFee`

### 3.3 Персистентность корзины
- [ ] Товары сохраняются после перезапуска приложения (AsyncStorage)
- [ ] Очистка корзины после успешного оформления заказа

## 4. Оформление заказа (Checkout)

### 4.1 Валидация перед заказом
- [ ] Без выбранного адреса → Alert «Выберите адрес доставки»
- [ ] Без авторизации → редирект на `/login`
- [ ] Пустая корзина → кнопка «Оформить» неактивна или скрыта

### 4.2 Процесс оформления
- [ ] Выбор адреса доставки из сохранённых
- [ ] Выбор способа оплаты (наличные / онлайн)
- [ ] Клик «Оформить заказ» → создаётся запись в `orders` + позиции в `order_items`
- [ ] `totalAmount` в заказе совпадает с `totalPrice` из корзины
- [ ] Корзина очищается после успешного заказа
- [ ] Редирект на экран истории заказов `/orders`

### 4.3 Push-уведомления (нативная платформа)
- [ ] После заказа приходит уведомление «Заказ оформлен» (~2 сек)
- [ ] Через ~15 сек приходит уведомление «Курьер в пути»
- [ ] На Web — Alert вместо push-уведомлений

### 4.4 Обработка ошибок
- [ ] Ошибка сети при создании заказа → Alert с текстом ошибки
- [ ] `isSubmitting` корректно сбрасывается в `finally` (кнопка разблокируется)

## 9. Кросс-платформенность

### 9.1 Android
- [ ] Тени отображаются через `elevation` (не `shadow*`)
- [ ] Multiline `TextInput` с `textAlignVertical: 'top'`
- [ ] `KeyboardAwareScrollView` с `enableOnAndroid` в формах

### 9.2 iOS
- [ ] Тени через `shadow*` свойства
- [ ] `SafeAreaView` — контент не залезает под вырез/статусбар
- [ ] Шрифты: SF system-ui

### 9.3 Web
- [ ] SSR guard — `localStorage` не вызывается на сервере
- [ ] `window.alert()` вместо `Alert.alert()` где нужно
- [ ] Push-уведомления не вызываются (нет `schedulePushNotification`)

## 10. Общие проверки качества

### 10.1 Производительность
- [ ] Нет заметных лагов при скролле списков (FlatList с `keyExtractor`)
- [ ] Изображения оптимизированы (нет загрузки full-res для превью)

### 10.2 Обработка ошибок
- [ ] Нет крашей при отсутствии сети — graceful degradation
- [ ] Нет крашей при невалидных данных от Supabase (null checks)
- [ ] `ErrorBoundary` ловит неожиданные ошибки рендера

### 10.3 Стили и тема
- [ ] Нет raw hex-цветов в компонентах (только `Colors.light.*`)
- [ ] Все отступы через `Spacing.*`, радиусы через `Radius.*`
- [ ] Тёмная тема: все экраны корректно переключаются (если поддерживается)

### 10.4 Автоматические проверки
- [ ] `npm run lint` — 0 warnings
- [ ] `npm run type-check` — 0 ошибок
- [ ] `npm test` — все тесты проходят

# Smoke Tests / План ручных проверок

> Используйте после каждого крупного изменения (рефакторинг, новые фичи, обновление зависимостей).
> Отмечайте статус `[PASS]` / `[FAIL]` / `[SKIP]`.

---

## 1. Авторизация и Профиль

### 1.1 Вход по номеру телефона
- [ ] Экран логина открывается без ошибок
- [ ] Ввод номера телефона → получение SMS с OTP
- [ ] Ввод корректного OTP → редирект на главный экран (Home)
- [ ] Ввод неверного OTP → отображение ошибки, повторная попытка доступна
- [ ] Таймер повторной отправки OTP (кнопка «Отправить повторно») блокируется на время ожидания

### 1.2 Состояние авторизации
- [ ] После перезапуска приложения пользователь остается залогиненным
- [ ] Логаут (выход из профиля) → корректный разлогин + редирект на Login
- [ ] `AuthProvider` корректно обрабатывает `loading` в `false` (нет бесконечного спиннера)
- [ ] Профиль пользователя автоматически синхронизируется с таблицей `profiles` после входа

### 1.3 Выход из аккаунта
- [ ] Кнопка «Выйти» в профиле → запрос подтверждения, редирект на логин
- [ ] После выхода доступ к защищенным экранам закрыт (редирект на логин)

## 3. Корзина и Ценообразование

### 3.1 Управление корзиной
- [ ] Добавление товара → счетчик появляется в корзине, сумма обновляется
- [ ] Изменение количества → `totalItems` и `subtotal` пересчитываются мгновенно
- [ ] Уменьшение количества до 0 → товар удаляется из корзины
- [ ] Жест удаления (swipe) в корзине работает (если реализовано)
- [ ] Batch-обновление (синхронизация с БД) работает без ошибок и лагов

### 3.2 Расчет суммы и доставки
- [ ] `subtotal` = Сумма(Цена * Количество) по всем позициям
- [ ] Корзина < 700₽ → `deliveryFee` = 90₽ (или актуальная сумма)
- [ ] Корзина ≥ 700₽ → `deliveryFee` = 0₽
- [ ] Пустая корзина → `deliveryFee` = 0₽, `totalPrice` = 0₽
- [ ] `totalPrice` = `subtotal` + `deliveryFee`

### 3.3 Персистентность корзины
- [ ] Корзина сохраняется после перезапуска приложения (AsyncStorage)
- [ ] Очистка корзины после успешного оформления заказа

## 4. Оформление заказа (Checkout)

### 4.1 Переход к чекауту
- [ ] При попытке без адреса → Alert «Выберите адрес доставки»
- [ ] Без авторизации → редирект на `/login`
- [ ] Пустая корзина → кнопка «Оформить» неактивна или скрыта

### 4.2 Процесс оформления
- [ ] Выбор адреса доставки из сохраненных или ввод нового
- [ ] Выбор способа оплаты (Картой / Наличными)
- [ ] Кнопка «Оформить заказ» → создание записи в `orders` + позиции в `order_items`
- [ ] `totalAmount` в БД совпадает with `totalPrice` в корзине
- [ ] Корзина очищается после успешного оформления
- [ ] Редирект на экран успеха или список заказов `/orders`

### 4.3 Push-уведомления (при наличии)
- [ ] После оформления приходит уведомление «Заказ принят» (~2 сек)
- [ ] Через ~15 сек приходит уведомление «Заказ в пути» (тест триггера)
- [ ] В Web-версии — Alert вместо push-уведомления

### 4.4 Обработка ошибок
- [ ] Ошибка при создании заказа → Alert с текстом ошибки
- [ ] `isSubmitting` корректно сбрасывается в `finally` (кнопка разблокируется)

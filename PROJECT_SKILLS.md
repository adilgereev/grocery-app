# 🎯 Манифест Разработки Grocery App

Этот документ содержит ключевые принципы и «скиллы», на которые опирается AI-ассистент при работе над проектом. Каждый запрос проходит через этот фильтр.

---

## 1. 📊 Ориентир на Продаваемость (Marketability)
*Приложение должно не просто работать, а ПРОДАВАТЬ.*
- **Визуальное превосходство:** Использовать премиальные тени, мягкие градиенты и качественные изображения. Если нет фото — использовать заглушки с приятными цветами.
- **Акценты на главном:** Цены и кнопки «В корзину» должны быть самыми заметными элементами.
- **Маркетинговые фишки:** Внедрять бейджи («TOP», «SALE»), динамические баннеры и персонализированные приветствия.
- **Скорость:** Никаких лишних загрузок. Пользователь не должен ждать, пока обновится адрес или добавится товар.

## 2. 🎨 Дизайн-система и Токены (UI/UX Skill)
- **Строгое следование `theme.ts`:** Все отступы только через `Spacing`, радиусы через `Radius`, цвета через `Colors`.
- **Никаких «Magic Numbers»:** Если нужно 10px, а в токенах есть 8 (`s`) и 16 (`m`), выбираем ближайший токен или обосновываем расширение системы.
- **Анимации:** Мягкие переходы, `Animated` для шапок и списков. Интерфейс должен «дышать».

## 3. 🏗 Архитектура и Реактивность (Core App Skill)
- **Zustand — источник правды:** Состояние (корзина, адреса, профиль) хранится в сторах. Компоненты реактивно на него подписываются.
- **Supabase как Backend:** Использовать возможности PostgreSQL на максимум (фильтрация, сортировка на стороне БД).
- **Безопасность (RLS):** Всегда проверять политики доступа при создании новых таблиц.

## 4. 📂 Инфраструктура Storage
- **Порядок в медиа:** Использовать папки `/products/` и `/categories/`. 
- **Оптимизация:** Грузить только те форматы и размеры, которые нужны для мобильных экранов.

## 5. 🇷🇺 Кодекс Кода
- **Комментарии:** Только на русском языке.
- **Чистота:** Удалять неиспользуемые импорты и логи.
- **Документация:** Поддерживать `BACKLOG.md` и `walkthrough.md` в актуальном состоянии.

## 6. 📱 Мультиплатформенность (Android + iOS)
- **Универсальные тени:** Всегда дублировать `shadow`-свойства (iOS) свойством `elevation` (Android) для сохранения глубины интерфейса.
- **Безопасные зоны (Safe Areas):** Использовать `SafeAreaView` и избегать жестких `top`/`bottom` отступов, которые могут перекрываться статус-баром или "челкой".
- **Работа с клавиатурой:** Проверять поведение полей ввода на обеих ОС. Использовать `enableOnAndroid` для `KeyboardAwareScrollView`.
- **Инпуты:** Всегда добавлять `textAlignVertical: 'top'` для многострочных полей (Android) и проверять внутренние паддинги.
- **Шрифты и иконки:** Использовать системные шрифты через токены, чтобы текст выглядел нативно и на San Francisco (iOS), и на Roboto (Android).

## 7. 🛡️ Работа со слоями (Layers & Z-Index)
- **Приоритет выпадающих списков:** Любой компонент с выпадающим списком (поиск, фильтры) должен иметь `zIndex` выше, чем у последующих элементов в JSX.
- **Z-Index у родителей:** Помнить, что в React Native `zIndex` дочернего элемента ограничен контекстом родителя. Если Card 1 должна перекрывать Card 2, `zIndex` нужно ставить на саму Card 1.
- **Elevation (Android):** Для Android всегда сочетать `zIndex` с `elevation`, так как `elevation` влияет на визуальный порядок слоев.
- **Порталы (Portals):** Для сложных глобальных оверлеев рассматривать возможность выноса в `Portal` на уровень корня приложения.

## 8. 💎 Стиль Emerald Minimalism (Clean & Premium UI)
*Приложение должно выглядеть как премиальный цифровой продукт.*
- **Принципы Clean UI:** Удалять весь «информационный шум».
- **Премиальные тени:** Использовать максимально мягкие тени (`shadowOpacity: 0.04-0.08`, `shadowRadius: 10-14`) для создания эффекта глубины и «стеклянного» объема.
- **Радиусы:** Приоритет крупным скруглениям (`Radius.l` — обычно 20px) для карточек и основных кнопок.
- **Акценты и Состояния:** Главный цвет — Изумрудный (`#10B981`). Выбранные элементы подсвечивать мягким светло-зеленым фоном (`#F0FDF4`) с акцентной рамкой.
- Используйте `lucide-react-native` для иконок.
- Все ассеты (изображения) должны быть типизированы или находиться в константах.

## 🧪 Стандарты тестирования (Jest)

Для обеспечения безопасности кода, особенно при разработке через ИИ, мы используем **Jest** и **React Native Testing Library**.

1. **Unit-тесты (Логика)**:
   - Любая сложная функция трансформации данных (цены, адреса, даты) **ОБЯЗАТЕЛЬНО** должна иметь `.test.ts`.
   - Проверяйте крайние случаи: `null`, `undefined`, пустые строки, некорректные типы.

2. **UI-тесты (Компоненты)**:
   - Критические UI-компоненты (карточки товара, кнопки корзины, формы) должны иметь базовый тест на рендеринг.
   - Используйте `testID="имя-компонента"` для надежного поиска элементов в тестах. **НИКОГДА** не полагайтесь на поиск по тексту, если это не обосновано логикой теста.
   - Используйте `StyleSheet.flatten(element.props.style)` для проверки стилей.

3. **Запуск**: Перед каждым коммитом запускайте `npm test`.

## 9. 🗄️ Управление Базой Данных (Database Management)
- **Строгое следование Workflow:** При любых изменениях в структуре БД (таблицы, RLS, функции, типы) всегда руководствоваться процессом из [SUPABASE_WORKFLOW.md](file:///d:/Dev/JS%20projects/grocery-app/SUPABASE_WORKFLOW.md).
- **Единый источник правды:** Файл `supabase/schema.sql` должен всегда содержать актуальный и полный дамп всей базы данных для мгновенного развертывания.
- **Миграции через CLI:** Использовать `npx supabase migration new` для фиксации изменений, прежде чем обновлять основной файл схемы.

## 10. 🧹 Правило Бойскаута (Boy Scout Rule)
*Оставляй код чище, чем он был до тебя.*
- Если вы зашли в файл для исправления бага или добавления фичи — исправьте по пути мелкие огрехи линта (инлайновые стили, HEX-коды, неиспользуемые импорты).
- Не откладывайте «причесывание» кода на потом; чистота поддерживается инкрементально.

## 11. 🛠 Технические Стандарты Качества (Quality & Logic)
Для того чтобы проект оставался стабильным и предсказуемым, мы следуем этим правилам:

1. **Мемоизация и Зависимости (`exhaustive-deps`):**
   - Все функции, которые используются внутри `useEffect` или передаются глубоко в дочерние компоненты, **ОБЯЗАТЕЛЬНО** должны быть обернуты в `useCallback`.
   - Это гарантирует стабильность ссылок и предотвращает бесконечные циклы обновлений.

2. **Цветовые Токены (`no-color-literals`):**
   - Использование HEX-кодов (напр. `#ffffff`, `#000`) или `rgba()` напрямую в компонентах **ЗАПРЕЩЕНО**.
   - Всегда используйте токены из `Colors.light.` (например, `Colors.light.card`, `Colors.light.text`).
   - Это обеспечивает консистентность дизайна и позволяет легко внедрять темную тему в будущем.

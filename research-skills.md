# Research: External Skills for Claude Code

Список готовых скиллов из интернета, которые могли бы быть полезны для проекта Grocery App.

## 🔥 Высокий приоритет

### 1. **Supabase Schema & RLS Expert**
- **Назначение**: Автоматизация миграций, анализ RLS-политик, оптимизация запросов
- **Почему нужен**: У проекта строгий RLS на каждой таблице; миграции требуют синхронизации типов после каждого изменения
- **Возможности**:
  - Проверка RLS-политик на новых таблицах
  - Валидация синхронизации `npm run supabase:types`
  - Рекомендации по индексам и оптимизации запросов
  - Автогенерация миграций из изменений схемы

### 2. **React Native Performance Profiler**
- **Назначение**: Анализ рендеров, оптимизации через `React.memo`, `useCallback`, batch updates
- **Почему нужен**: Мобильное приложение — FPS-дропы видны сразу; требуется строгое соблюдение `useCallback` и мемоизации
- **Возможности**:
  - Профилирование компонентов на лишние рендеры
  - Автопроверка `exhaustive-deps` в useEffect
  - Рекомендации по оптимизации Zustand селекторов
  - Анализ памяти и утечек

### 3. **Jest Test Generator & Coverage Analyzer**
- **Назначение**: Генерация тестов по коду, анализ покрытия, регрессионные проверки
- **Почему нужен**: Требование `npm test` перед каждым коммитом; AI Regression Cycle обязателен
- **Возможности**:
  - Автогенерация test skeletons для компонентов
  - Анализ coverage по компонентам и stores
  - Проверка что тесты остаются зелёными после изменений
  - Генерация mock-данных для Supabase

### 4. **TypeScript Strict Mode Enforcer**
- **Назначение**: Анализ типов, автоисправление слабых типизаций
- **Почему нужен**: Проект использует `strict: true`; нужна помощь в поиске лазеек в типизации
- **Возможности**:
  - Проверка на `any` и неявные `any`
  - Валидация типов из `types/supabase.ts`
  - Рекомендации по улучшению типизации props
  - Проверка discriminated unions для типов

## 🎯 Средний приоритет

### 5. **Expo Router Navigator**
- **Назначение**: Визуализация маршрутов, проверка `testID`, валидация стека
- **Почему нужен**: Сложные навигационные цепочки (addresses → manage-address); риск навигационных багов
- **Возможности**:
  - Визуализация дерева маршрутов
  - Проверка что экраны зарегистрированы в `_layout.tsx`
  - Валидация `testID` на всех navigable элементах
  - Анализ глубоких линков

### 6. **ImageKit & CDN Optimizer**
- **Назначение**: Анализ изображений, рекомендации по размерам, кеширование
- **Почему нужен**: Проект использует ImageKit + R2; нужны советы по оптимальным параметрам
- **Возможности**:
  - Проверка что `getOptimizedImage()` вызывается везде, где нужно
  - Рекомендации по `width`, `quality`, `dpr` для разных экранов
  - Валидация LQIP (placeholder) URLs
  - Анализ кеша и версионирования

### 7. **Zustand Store Debugger**
- **Назначение**: Визуализация стора, отслеживание мутаций, проверка селекторов
- **Почему нужен**: Помощь при debug состояния; проверка что селекторы оптимальны
- **Возможности**:
  - Мониторинг изменений в хранилище
  - Проверка что используются селекторы (не весь store)
  - Анализ persisted stores
  - Отслеживание утечек памяти в асинхронных операциях

### 8. **GitHub Actions CI/CD Setup**
- **Назначение**: Автоматизация lint → test → type-check перед мержем
- **Почему нужен**: Гарант качества при работе в команде; Pre-commit hook уже есть локально
- **Возможности**:
  - Автоматический lint + type-check на PR
  - Запуск тестов перед мержем
  - Проверка что BACKLOG.md обновлён
  - Отправка уведомлений при ошибках

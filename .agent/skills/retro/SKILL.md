# Retro: Еженедельная Ретроспектива

## 🎯 Цель

Понять что было сделано, где тратится больше всего времени, и что улучшить на следующей неделе. Только факты из git-истории — никаких домыслов.

---

## 📊 Сбор данных

```bash
# Коммиты за период (по умолчанию 7 дней)
git log --since="7 days ago" --oneline --author="Adilgereev"

# Хотспоты — файлы с наибольшим числом изменений
git log --since="7 days ago" --format="%H" | xargs -I{} git diff-tree --no-commit-id -r --name-only {} | sort | uniq -c | sort -rn | head -15

# Статистика по типам коммитов (conventional commits)
git log --since="7 days ago" --format="%s" --author="Adilgereev"

# Детальный diff для понимания масштаба
git log --since="7 days ago" --stat --author="Adilgereev" | tail -5
```

Для `/retro 14` и `/retro 30` — заменить `"7 days ago"` на `"14 days ago"` / `"30 days ago"`.

---

## 📋 Структура отчёта

### 1. 📦 Что было сдано
Список коммитов сгруппированный по типу:
- `feat:` — новые фичи
- `fix:` — исправления
- `refactor:` — рефакторинг
- `test:` — тесты
- `style:` / `chore:` — прочее

### 2. 🔥 Хотспоты
Топ-5 файлов по числу изменений.

**Интерпретация:**
- >5 изменений за неделю → кандидат на декомпозицию или нестабильная логика
- Один файл постоянно меняется → возможно, слишком много ответственности

### 3. 📈 Соотношение работ
```
feat  : ████████░░  N коммитов
fix   : ████░░░░░░  N коммитов
refactor: ██░░░░░░  N коммитов
test  : █░░░░░░░░░  N коммитов
```

Норма: feat > fix > refactor > test. Если fix > feat — много технического долга.

### 4. 📝 BACKLOG.md статус
- Прочитать BACKLOG.md
- Отметить что закрыто за период
- Что ещё открыто

### 5. 💡 1-2 улучшения на следующий период
Конкретные, измеримые. Не "писать больше тестов", а "добавить тест для [конкретного компонента]".

# UI Standards: Soft Minimalism

## 1. Ориентир на Продаваемость (Marketability)
*Приложение должно не просто работать, а ПРОДАВАТЬ.*
- **Визуальное превосходство**: Мягкие тени, экстремальное скругление, стеклянные эффекты. Если нет фото — использовать заглушки (`Skeleton`) с приятными цветами.
- **Акценты на главном**: Цены и кнопки «В корзину» — самые заметные элементы через pill-форму и контрастный фон.
- **Маркетинговые фишки**: Бейджи («TOP», «SALE»), динамические баннеры и персонализированные приветствия.
- **Скорость**: Никаких лишних загрузок. Пользователь не должен ждать.

## 2. Дизайн-система и Токены
- **Тема**: Использовать `theme.ts` для всех стилей. Никаких «Magic Numbers» (hex-коды, произвольные отступы).
- **Emerald Accents**: Основной цвет — `#10B981` (Emerald), но подача мягче и нейтральнее.
- **Soft Minimalism**: Чистый, дружелюбный UI с экстремальным скруглением (до pill) и очень мягкими тенями.
- **Shadows**: Нейтральные серые тени (`shadowColor: Colors.light.text`, `shadowOpacity: 0.03–0.05`, `shadowRadius: 12–16`). Никаких цветных теней.
- **Скругление**: Ключевой визуальный маркер. Карточки — `Radius.xxl (24)`, кнопки — pill-форма (`Radius.pill`), изображения — `Radius.xl (20)`.
- **Фоны**: Основной фон `#F9FAFB` (не чистый белый), карточки — `#ffffff`. Создаёт глубину без теней.
- **Типографика**: Использовать системные шрифты через токены. `fontWeight: '700'` — стандартный акцентный вес.

## 3. Ключевые паттерны Soft Minimalism
- **Pill-элементы**: Кнопки действий, инпуты, контролы количества — всё pill-формы.
- **Нейтральные тени**: Серые, не цветные. `elevation: 0` на Android, глубина только через shadow-свойства.
- **Soft Bold**: Акцентный текст через `fontWeight: '700'`, без экстремальных весов.
- **Щедрые отступы**: Много воздуха, сетка 8px, ничего не прижато вплотную.

## 4. Мультиплатформенность (Android + iOS)
- **Универсальные тени**: Дублировать `shadow`-свойства (iOS) свойством `elevation` (Android). В Soft Minimalism приоритет — мягкие shadow-свойства, elevation минимальный.
- **Безопасные зоны (Safe Areas)**: Использовать `SafeAreaView` из `react-native-safe-area-context` и избегать жестких `top`/`bottom` отступов.
- **Работа с клавиатурой**: Использовать `KeyboardAwareScrollView` (с `enableOnAndroid`) для полей ввода.
- **Инпуты**: Обязательно добавлять `textAlignVertical: 'top'` для многострочных полей (Android). Pill-форма обязательна.

## 5. Работа со слоями (Layers & Z-Index)
- **Приоритет выпадающих списков**: Компоненты с выпадающими элементами (поиск, фильтры) должны иметь `zIndex` выше, чем у последующих элементов в JSX.
- **Z-Index у родителей**: Помнить, что в React Native `zIndex` дочернего элемента ограничен контекстом родителя.
- **Elevation (Android)**: Всегда сочетать `zIndex` с `elevation`, так как `elevation` влияет на визуальный порядок слоев.

## 6. Переиспользуемые UI-компоненты
- **HeaderHandling**: Для всех stack-экранов ОБЯЗАТЕЛЬНО использовать `<ScreenHeader />` внутри `<SafeAreaView edges={['bottom']}>`. Хардкодить шапки вручную запрещено.
- **ProductCard**: Унифицированная карточка товара. Скругление `Radius.xxl`, мягкие тени, pill-кнопка добавления.
- **Skeleton**: Единые состояния загрузки.

## 7. Визуальные отличия от предыдущего стиля (Emerald Minimalism)
| Характеристика | Emerald Minimalism (было) | Soft Minimalism (стало) |
|---|---|---|
| Тени | Цветные emerald, opacity 0.04–0.08 | Нейтральные серые, opacity 0.03–0.05 |
| Скругление | Стандартное 8–16px | Экстремальное 20–24px, pill |
| Фон | Чистый белый (#fff) | Тёплый off-white (#F9FAFB) |
| Кнопки | Solid primary color | Primary light фон (#F0FDF4), pill-форма |
| Elevation | Классическое Android elevation | elevation: 0, глубина через shadow |

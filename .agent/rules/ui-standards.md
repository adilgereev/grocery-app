# UI Standards: Soft Minimalism

## 1. Ориентир на Продаваемость (Marketability)
*Приложение должно не просто работать, а ПРОДАВАТЬ.*
- **Визуальное превосходство**: Мягкие тени, экстремальное скругление, стеклянные эффекты. Если нет фото — использовать заглушки (`Skeleton`) с приятными цветами.
- **Акценты на главном**: Цены и кнопки «В корзину» — самые заметные элементы через pill-форму и контрастный фон.
- **Маркетинговые фишки**: Бейджи («TOP», «SALE»), динамические баннеры и персонализированные приветствия.
- **Скорость**: Никаких лишних загрузок. Пользователь не должен ждать.

## 2. Дизайн-система и Токены
- **Тема**: Использовать `theme.ts` для всех стилей. Никаких «Magic Numbers» (hex-коды, произвольные отступы).
- **Emerald Accents**: Основной цвет — `#10B981` (Emerald), но подача мягче и нейтральнее.
- **Soft Minimalism**: Чистый, дружелюбный UI с экстремальным скруглением (до pill) и очень мягкими тенями.
- **Shadows**: Нейтральные серые тени. Всегда использовать спред-токены: `...Shadows.sm`, `...Shadows.md`, `...Shadows.lg`. Прямые `shadowColor` в компонентах **запрещены**. `elevation: 0` на Android.
- **Скругление**: Ключевой визуальный маркер. Карточки — `Radius.xxl (24)`, кнопки — pill-форма (`Radius.pill`), изображения — `Radius.xl (20)`. Круглые иконки/аватары любого размера — всегда `Radius.pill`.
- **Отступы**: Токены `Spacing.sm (12)` и `Spacing.ml (20)` добавлены и обязательны. Значения `18`, `30`, `36`, `6` — намеренные дизайн-константы (встречаются 1–2 раза, токен нецелесообразен), оставлять как есть.
- **Фоны**: Основной фон `#F9FAFB` (не чистый белый), карточки — `#ffffff`. Создаёт глубину без теней.
- **Типографика**: Использовать системные шрифты через токены. `fontWeight: '700'` — стандартный акцентный вес.

## 3. Ключевые паттерны Soft Minimalism
- **Pill-элементы**: Кнопки действий, инпуты, контролы количества — всё pill-формы.
- **Нейтральные тени**: Серые, не цветные. `elevation: 0` на Android, глубина только через shadow-свойства.
- **Soft Bold**: Акцентный текст через `fontWeight: '700'`, без экстремальных весов.
- **Щедрые отступы**: Много воздуха, сетка 8px, ничего не прижато вплотную.

## 4. Мультиплатформенность (Android + iOS)
- **Универсальные тени**: Дублировать `shadow`-свойства (iOS) свойством `elevation` (Android). В Soft Minimalism приоритет — мягкие shadow-свойства, elevation минимальный.
- **Безопасные зоны (Safe Areas)**: Использовать `SafeAreaView` из `react-native-safe-area-context` и избегать жестких `top`/`bottom` отступов.
- **Работа с клавиатурой**: Использовать `KeyboardAwareScrollView` (с `enableOnAndroid`) для полей ввода.
- **Инпуты**: Обязательно добавлять `textAlignVertical: 'top'` для многострочных полей (Android). Pill-форма обязательна.

## 5. Работа со слоями (Layers & Z-Index)
- **Приоритет выпадающих списков**: Компоненты с выпадающими элементами (поиск, фильтры) должны иметь `zIndex` выше, чем у последующих элементов в JSX.
- **Z-Index у родителей**: Помнить, что в React Native `zIndex` дочернего элемента ограничен контекстом родителя.
- **Elevation (Android)**: Всегда сочетать `zIndex` с `elevation`, так как `elevation` влияет на визуальный порядок слоев.

## 6. Переиспользуемые UI-компоненты
- **HeaderHandling**: Для всех stack-экранов ОБЯЗАТЕЛЬНО использовать `<ScreenHeader />` внутри `<SafeAreaView edges={['bottom']}>`. Хардкодить шапки вручную запрещено.
  - **Намеренные исключения без `<ScreenHeader />`**: `app/product/[id].tsx` (overlay-кнопки поверх fullscreen-фото, кастомный `ProductHeader`) и `app/setup-profile.tsx` (onboarding-стиль, декоративная шапка). Это стандарт — не исправлять.
- **ProductCard**: Унифицированная карточка товара. Скругление `Radius.xxl`, мягкие тени, pill-кнопка добавления.
  - **Намеренное исключение**: `app/(admin)/catalog.tsx` — кастомная строка с кнопками «Изменить/Удалить». Это admin-UI, не клиентская карточка покупки. `<ProductCard />` здесь не применять.
- **Skeleton**: Единые состояния загрузки.

## 7. Принципы цветового UX

### Монохромная палитра
Приложение использует монохромную изумрудную палитру — все акценты в одном цветовом семействе:
- `primaryLight` `#F0FDF4` — мятный фон, выделение выбранных элементов
- `primary` `#10B981` — бренд, навигация, интерактивные иконки
- `cta` `#059669` — кнопки главного действия ("В корзину", "Оформить заказ")
- `ctaDark` `#047857` — pressed-состояние CTA

Янтарный/контрастный акцент намеренно не используется: монохромная схема создаёт более чистый и профессиональный вид без цветового шума.

### Правило 60-30-10
Соотношение цветов в интерфейсе:
- **60%** — нейтральный фон (`background`, `card`, `borderLight`)
- **30%** — фирменный изумрудный (`primary` — навигация, иконки, интерактивные акценты)
- **10%** — CTA-акцент (`cta` тёмно-зелёный — только кнопки главного действия: "В корзину", "Оформить заказ")

### Один цвет — одна роль
Каждый цвет несёт ровно один смысл. Нельзя использовать один токен для двух разных смысловых контекстов (например, CTA и статусный бейдж не могут быть одного цвета).

### Иерархия внимания через цветовой вес
Чем важнее действие, тем насыщеннее цвет. Фоны — нейтральные, вторичный текст — серый, CTA — максимально насыщенный. Несколько одинаково насыщенных элементов на экране создают шум.

### Семантические цвета — только для статусов
`error`, `warning`, `info`, `success` — системный язык с устоявшимися ожиданиями пользователя. Использовать только для передачи смысла (ошибка, предупреждение, процесс, успех), **не для декора** иконок или плиток меню.

### Минимальный набор акцентов на экране
Не более 2 акцентных цветов одновременно на одном экране: 1 бренд-цвет + 1 CTA. Семантические цвета добавляются только при необходимости передать статус. Декоративное разнообразие цветов создаёт визуальный шум.

### Текущее распределение ролей в проекте
| Токен | Роль | Где применять |
|---|---|---|
| `primary` | Бренд, навигация, выбор | Иконки, подчёркивания, индикаторы, выделение строк |
| `cta` (#059669) / `ctaLight` | Главное действие | Только "В корзину", "Оформить заказ", FAB |
| `warning` | Требует внимания | Статус "Новый заказ", предупреждения |
| `info` | В процессе | Статусы "Сборка", "В пути", информационные тосты |
| `success` | Завершено | Статус "Доставлен", успешные тосты |
| `error` | Ошибка/опасность | Статус "Отменён", ошибки валидации |

## 8. Визуальные отличия от предыдущего стиля (Emerald Minimalism)
| Характеристика | Emerald Minimalism (было) | Soft Minimalism (стало) |
|---|---|---|
| Тени | Цветные emerald, opacity 0.04–0.08 | Нейтральные серые, opacity 0.03–0.05 |
| Скругление | Стандартное 8–16px | Экстремальное 20–24px, pill |
| Фон | Чистый белый (#fff) | Тёплый off-white (#F9FAFB) |
| Кнопки | Solid primary color | Primary light фон (#F0FDF4), pill-форма |
| Elevation | Классическое Android elevation | elevation: 0, глубина через shadow |

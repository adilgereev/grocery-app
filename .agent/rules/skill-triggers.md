# Skill Triggers: Когда и Как Запускать Скиллы

Этот файл — обязательная инструкция. При каждом запросе пользователя сверяться с таблицей триггеров ниже и действовать согласно колонке «Действие».

---

## 🔴 Автоматически (без вопросов)

| Условие | Скилл | Действие |
|---------|-------|----------|
| Пользователь описывает баг, краш, ошибку, неожиданное поведение | `/investigate` | Запустить Skill("investigate") **до** написания любого кода |
| Пользователь просит "проверь код", "посмотри на", "review", перед коммитом/мержем | `/review` | Запустить Skill("review") |
| Пользователь просит "проверь дизайн", "аудит UI", после правок компонентов/экранов | `/design-review` | Запустить Skill("design-review") |

---

## 🟡 Предложить (одна строка в конце ответа)

| Условие | Скилл | Формат предложения |
|---------|-------|--------------------|
| Завершена реализация фичи или исправление бага | `/review` | "Запустить `/review` перед мержем?" |
| Создан или изменён UI-компонент / экран | `/design-review` | "Запустить `/design-review` для проверки токенов?" |
| Исправлен баг (после `/investigate`) | `/review` | "Запустить `/review` чтобы проверить смежный код?" |
| Пользователь упоминает "неделя", "итоги", "что сделано", "прогресс" | `/retro` | "Запустить `/retro` для анализа прогресса?" |
| Пятница или конец рабочего дня (если упомянуто) | `/retro` | "Запустить `/retro` — итоги недели?" |

---

## ⚪ Не предлагать (когда НЕ нужно)

- `/investigate` — если пользователь уже знает причину и просит реализацию
- `/review` — если изменение тривиальное (переименование, одна строка)
- `/design-review` — если изменение только в логике, без StyleSheet
- `/retro` — если разговор про конкретную задачу, не про прогресс

---

## 📌 Правила приоритета

1. Красные триггеры — абсолютный приоритет. Запустить скилл до любых других действий.
2. Жёлтые триггеры — одно предложение в конце ответа, не раньше.
3. Не предлагать больше одного скилла за раз — не стекать несколько предложений.
4. Если пользователь уже явно назвал скилл (`/investigate`, `/retro` и т.д.) — не предлагать повторно, запустить.

# 🧪 Тестирование и Next-Gen Workflow

Этот документ определяет стандарты качества и процессы автоматизированного тестирования в Grocery App.

## 1. Стандарты тестирования (Jest)

Для обеспечения стабильности приложения и предотвращения регрессий мы используем **Jest** и **React Native Testing Library**.

### 1.1 Инфраструктура и Моки
- Все общие моки (`AsyncStorage`, `Reanimated`, `Expo Router`, `Supabase`) хранятся в [jest.setup.js](file:///d:/Dev/JS%20projects/grocery-app/jest.setup.js). **ЗАПРЕЩЕНО** дублировать их в каждом тестовом файле.
- Если тесту нужен специфичный мок, используйте `jest.mock()` внутри файла, но проверяйте, нет ли уже готового решения в `setup`.

### 1.2 Тестируемость через Декомпозицию
- Если элемент списка или сложный блок UI находится внутри экрана (inline), его **ОБЯЗАТЕЛЬНО** нужно вынести в `components/` перед написанием тестов.
- Экраны (`app/*.tsx`) тестировать крайне сложно из-за множества провайдеров, поэтому упор делаем на **Unit-тесты сторов** и **Component-тесты UI**.

### 1.3 UI-тесты (Компоненты)
- Используйте `testID` для всех интерактивных элементов (кнопки, инпуты, карточки).
- Проверяйте не только рендеринг, но и **взаимодействие** через `fireEvent.press`, `fireEvent.changeText`.
- Для проверки стилей (ширина, цвета) используйте `StyleSheet.flatten(element.props.style)`.

### 1.4 Unit-тесты (Логика)
- Сторы Zustand должны быть покрыты тестами на 100% логики (добавление, удаление, расчеты).
- Любая сложная функция трансформации (адреса, даты) должна иметь `.test.ts`.

## 2. 🤖 Next-Gen AI Workflow (Процесс разработки с ИИ)

Для достижения максимального качества кода при работе с ИИ ассистентом, мы придерживаемся следующего цикла:

1. **Тесты как Контракт:** Перед внесением сложных изменений ИИ анализирует существующие тесты или пишет новые. Текст теста — это самое точное техническое задание.
2. **AI Регрессия:** После каждого изменения в логике или UI, ИИ **ОБЯЗАН** запустить соответствующие тесты через `npm test`. Это гарантирует, что новые фичи не сломали старый функционал (например, правка в `addressStore` не затронула `cartStore`).
3. **Самокоррекция:** Если тест падает, ИИ анализирует ошибку и исправляет код или моки самостоятельно до того, как представить результат пользователю.
4. **Чистота тестов:** При создании новых компонентов ИИ сразу добавляет `testID` и базовые тесты на взаимодействие, делая проект «прозрачным» для будущих доработок.

## 3. Запуск
Перед каждым коммитом или крупным изменением запускайте:
```bash
npm test
```
Для проверки покрытия используйте:
```bash
npm test -- --coverage
```
